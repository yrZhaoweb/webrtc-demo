# 快速测试清单 ✅

## 🚀 5 分钟快速验证

### 步骤 1: 启动服务 (1 分钟)

```bash
cd webrtc-demo

# 方法 A: 使用自动化脚本（推荐）
chmod +x test-*.sh
./test-setup.sh  # 首次运行
./test-run.sh    # 启动服务

# 方法 B: 手动启动
# 终端 1
cd server && npm install && npm run dev

# 终端 2
cd web && npm install && npm run dev
```

### 步骤 2: 基本功能测试 (2 分钟)

#### 测试 1: 创建房间

- [ ] 打开浏览器访问 `http://localhost:5173`
- [ ] 点击"创建房间"
- [ ] 输入名字"测试用户A"
- [ ] 确认进入房间
- [ ] 检查右上角连接状态显示"已连接"（绿色）

#### 测试 2: 加入房间

- [ ] 打开新的浏览器窗口（或隐私模式）
- [ ] 复制第一个窗口的邀请链接
- [ ] 粘贴到新窗口访问
- [ ] 输入名字"测试用户B"
- [ ] 确认加入成功
- [ ] 两个窗口都显示 2 个在线用户

#### 测试 3: 发送消息

- [ ] 在窗口 A 发送消息"Hello from A"
- [ ] 窗口 B 收到消息
- [ ] 在窗口 B 发送消息"Hello from B"
- [ ] 窗口 A 收到消息

### 步骤 3: 连接状态验证 (1 分钟)

#### 检查连接状态

- [ ] 点击右上角的连接状态按钮
- [ ] 展开详细信息
- [ ] 确认显示：
  - 信令服务器: 已连接（绿色）
  - 对等连接: 1 个
  - 连接状态: connected
  - 数据通道状态: open

### 步骤 4: 重连测试 (1 分钟)

#### 测试自动重连

- [ ] 在浏览器开发者工具中打开 Network 标签
- [ ] 选择 "Offline" 模式
- [ ] 观察连接状态变为"重连中"（橙色）
- [ ] 取消 "Offline" 模式
- [ ] 观察自动重连成功
- [ ] 连接状态恢复为"已连接"（绿色）

## 🐛 如果测试失败

### 问题 1: 无法连接到服务器

**症状**: 连接状态一直显示"连接中"

**检查**:

```bash
# 检查服务器是否运行
curl http://localhost:8080
# 或
lsof -i :8080
```

**解决**:

- 确认服务器已启动
- 检查端口是否被占用
- 查看服务器日志: `tail -f logs/server.log`

### 问题 2: WebRTC 连接失败

**症状**: 用户列表显示但无法发送消息

**检查**:

- 打开浏览器控制台查看错误
- 访问 `chrome://webrtc-internals/` 查看详细信息
- 检查连接状态详情中的对等连接状态

**解决**:

- 确认防火墙未阻止 WebRTC
- 尝试使用不同的浏览器
- 检查网络环境（某些企业网络可能阻止 WebRTC）

### 问题 3: 消息发送失败

**症状**: 消息发送后对方收不到

**检查**:

- 查看连接状态中的数据通道状态
- 检查浏览器控制台错误
- 确认两个用户都在同一房间

**解决**:

- 刷新页面重新建立连接
- 检查网络连接
- 查看服务器日志
